# Enterprise Finance Knowledge Graph Explorer

I built this project to understand how finance data can be connected like a network.

Instead of looking at only normal tables, I wanted to see how vendors, cost centers, profit centers, GL accounts, and transactions are related to each other.

## What I did

In this project, I created:

- finance master data
- finance transactions
- relationship tables between finance entities
- a finance knowledge graph
- important node analysis
- impact analysis
- cluster detection
- relationship search

## Why I did this project

I did this project to learn how finance data can be analyzed in a better way than normal reporting.

I wanted to understand:

- which entities are strongly connected
- which vendors have more impact
- which cost centers and GL accounts are linked
- how groups of finance activity can be found
- how to search relationships using simple questions

## Main data used

This project uses:

- Vendors
- Cost Centers
- Profit Centers
- GL Accounts
- Finance Transactions

## What this project shows

This project shows how to:

- create finance data
- join finance entities
- build relationship tables
- create a graph from finance data
- find important connected nodes
- find clusters
- search finance relationships
